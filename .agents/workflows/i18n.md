# LOKMA Admin Portal i18n Workflow

## Supported Languages (7)
All i18n work MUST include translations for ALL 7 languages:

| Code | Language |
|------|----------|
| `de` | Deutsch (German) — Primary |
| `en` | English |
| `es` | Español (Spanish) |
| `fr` | Français (French) |
| `it` | Italiano (Italian) |
| `nl` | Nederlands (Dutch) |
| `tr` | Türkçe (Turkish) |

## Translation Files Location
```
admin_portal/messages/
├── de.json
├── en.json
├── es.json
├── fr.json
├── it.json
├── nl.json
└── tr.json
```

## Steps

### 1. Identify the namespace
Each page uses a namespaced `useTranslations()` hook. Find the namespace:
```tsx
const t = useTranslations('AdminAccount'); // namespace = AdminAccount
```

### 2. Replace hardcoded strings with `t('key_name')`
- Use snake_case for keys (following existing convention)
- Keep key names descriptive
- For strings with interpolation, use `t('key', { variable })` pattern

### 3. Handle constants outside components
Constants defined outside React components (e.g., `STATUS_OPTIONS`, `CATEGORY_MAPPING`) cannot use `t()` directly since it's a hook. Solution patterns:
- **Pattern A**: Move the constant inside the component
- **Pattern B**: Create a function: `const getStatusLabels = (t: any) => ({...})`
- **Pattern C**: Use key-only constants and resolve labels in JSX: `t(status.labelKey)`

### 4. Add keys to ALL 7 language JSON files
// turbo-all
Every new key MUST be added to all 7 JSON files under the correct namespace:
```json
{
  "AdminAccount": {
    "new_key": "Translation in this language"
  }
}
```

### 5. Verify
```bash
cd admin_portal && npm run build
```
Check for missing key warnings. Test in at least 2 locales.

## Important Rules
- **NEVER** replace a hardcoded string in one language with a hardcoded string in another language
- **ALWAYS** use `t('key')` and add to ALL 7 JSON files
- **Date locales**: Use the user's locale from `useLocale()` for `toLocaleDateString()`, not hardcoded `'de-DE'` or `'tr-TR'`
